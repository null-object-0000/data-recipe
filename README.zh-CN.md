# AI 有数 / DataRecipe

**操作一次页面，生成 AI Agent 可复用的数据配方。**

AI 有数可以将网页查询、API 响应、查询条件、分页规则和字段含义沉淀为一份可复用的数据配方。
以后无论在哪个 AI Agent 中，都可以复用这份配方来自动取数、整理和分析。

## 为什么做这个项目

很多网页和内部后台系统都有数据，但这些数据通常藏在查询表单和接口响应里。

非技术人员想做数据分析时，经常会遇到这些问题：

* 页面只能一页一页查，不能灵活汇总；
* 后台没有完整的数据统计能力；
* 临时取数依赖研发或数据同学；
* AI Agent 想使用这些数据，但不知道从哪里取、怎么取；
* 同一个查询动作每次都要人工重复操作。

AI 有数希望解决这个问题：

> 你只需要操作一次页面，AI 以后就知道怎么取数、整理和分析。

## 什么是数据配方

数据配方是一份 AI Agent 可以复用的说明书。

它描述了：

* 数据来自哪里；
* 如何查询数据；
* 支持哪些查询条件；
* 如何自动翻页获取更多数据；
* 返回字段分别是什么意思；
* 这些数据适合做哪些汇总和分析；
* 如何与其他页面或系统的数据关联。

它不是一次性的网页采集结果，而是一种可以长期复用的数据能力。

## 核心思路

AI 有数优先通过 API 响应获取结构化数据，而不是直接从 DOM 页面中抠数据。

DOM 可以用来辅助理解：

* 哪个按钮触发了查询；
* 哪些表单字段对应查询条件；
* 哪些表格列对应返回字段；
* 分页按钮如何影响接口参数。

但最终沉淀下来的核心能力，是基于 API 响应的数据配方。

## 目标能力

第一阶段会聚焦在：

1. 探测当前页面的 API 请求；
2. 识别可能有价值的数据接口；
3. 解析请求参数、响应字段和分页规则；
4. 生成可复用的数据配方；
5. 在本地使用当前登录状态重新执行数据查询；
6. 让 AI Agent 基于数据配方完成取数、汇总和分析。

## 不是做什么

AI 有数不是普通爬虫，也不是反爬绕过工具。

我们不做：

* 绕过登录权限；
* 破解验证码；
* 破解动态签名；
* 绕过风控或访问限制；
* 高频抓取；
* 未授权的数据采集。

AI 有数只面向用户有权访问的数据页面，并将用户已经可以查询的数据能力沉淀为可复用的数据配方。

## 适合场景

* 内部后台数据查询；
* 运营数据汇总；
* 产品临时取数分析；
* 多页面数据组合分析；
* 公开页面低频数据整理；
* 为 AI Agent 生成可复用的数据工具；
* 将老系统的列表查询能力转化为 AI 可用能力。

## 示例

用户在订单列表页操作一次：

1. 选择时间范围；
2. 点击查询；
3. 翻到下一页；
4. AI 有数识别背后的订单查询接口；
5. 生成一份「订单数据配方」。

以后用户可以直接问 AI：

* 帮我统计上周 GMV；
* 按项目汇总订单数和销售额；
* 找出退款率异常的项目；
* 导出最近 7 天已支付订单；
* 结合项目列表和预约列表分析转化率。

## 项目状态

项目仍处于早期探索阶段。

当前重点是验证：

* API 探测是否足够稳定；
* 数据接口识别是否准确；
* 数据配方格式是否清晰；
* AI Agent 是否能基于数据配方完成真实分析任务。

## 本地运行 MVP

### 安装依赖

```bash
pnpm install
```

### 构建浏览器插件

```bash
pnpm build:extension
```

构建产物在：

```text
apps/extension/dist
```

### 在 Chrome 中加载插件

1. 打开 `chrome://extensions`；
2. 开启「开发者模式」；
3. 点击「加载已解压的扩展程序」；
4. 选择 `apps/extension/dist`；
5. 点击浏览器工具栏里的「AI 有数」图标打开侧边栏。

### 手动验证

1. 打开 `docs/test-page.html`；
2. 在 AI 有数侧边栏点击「开始发现」；
3. 在测试页点击「触发 fetch 查询」或「触发 XHR 查询」；
4. 侧边栏应显示发现的数据来源数量、URL、请求方式、状态、查询条件和返回预览；
5. 展开「高级信息」可以查看最小数据配方草稿 JSON。

也可以在任意你有权访问的网页上点击「开始发现」，然后触发页面查询。当前 MVP 只做低频本地探测，不会绕过登录、验证码、风控或动态签名。

### 开发模式

```bash
pnpm dev:extension
```

该命令会监听并重新构建 `apps/extension/dist`。Chrome 扩展页面中需要手动点击刷新插件后再验证最新代码。

### 导出 Data Skill Package

完成一次发现后，可以在侧边栏中：

1. 为数据技能填写名称和用途说明；
2. 确认或修改返回字段名称；
3. 查看「试运行结果」确认能读取数据；
4. 在「技能包预览」中确认已包含必需文件；
5. 点击「导出技能包」。

当前 MVP 会下载一个 `.data-skill.json` 文件。它是临时的文本包格式，里面包含：

```text
packageName
files[]
  SKILL.md
  recipe.json
  examples.md
  README.md
```

后续会把这个导出格式升级为真实文件夹或 zip。当前阶段先用 JSON 文本包验证「可生成、可测试、可导出」的闭环。

### Data Skill Package 验收点

导出的技能包至少应满足：

* 包含 `SKILL.md`、`recipe.json`、`examples.md`、`README.md`；
* `SKILL.md` 能说明这个数据技能适合完成什么任务；
* `recipe.json` 包含数据来源、查询条件、返回字段和测试运行所需信息；
* 侧边栏「试运行结果」可以展示数据预览；
* 导出前没有提示缺少必需文件。
